# Função ONS_2_11 — Injeção de Corrente Reativa sob Defeito

## Código Completo

```matlab
function [id_ref, iq_ref, fault_flag] = ONS_2_11(Vpcc_pu, id_nom, Imax, Vnom)

    V_mag = Vpcc_pu;

    % Ganhos definidos pelos pontos da curva ONS
    k_low  = (0 - (-1)) / (0.85 - 0.5);   % ≈ 2.857  [pu_Iq / pu_V]
    k_high = (1 - 0)    / (1.2  - 1.1);   % = 10     [pu_Iq / pu_V]

    % Interceptos (passam pelos pontos de fronteira V=0.85 e V=1.1)
    b_low  = -k_low  * 0.85;   % ≈ -2.4286
    b_high = -k_high * 1.1;    % = -11

    if V_mag < 0.85
        % SUBTENSÃO: injeta corrente reativa capacitiva
        fault_flag = 1;
        iq_ref_raw = k_low * V_mag + b_low;
        iq_ref = max(min(iq_ref_raw, Imax), -Imax);
        id_max_available = sqrt(max(Imax^2 - iq_ref^2, 0));
        id_ref = min(id_nom, id_max_available);

    elseif V_mag > 1.1
        % SOBRETENSÃO: absorve corrente reativa
        fault_flag = 1;
        iq_ref_raw = k_high * V_mag + b_high;
        iq_ref = max(min(iq_ref_raw, Imax), -Imax);
        id_max_available = sqrt(max(Imax^2 - iq_ref^2, 0));
        id_ref = min(id_nom, id_max_available);

    else
        % REGIÃO NORMAL (0.85 ≤ V ≤ 1.1 pu)
        fault_flag = 0;
        id_ref = id_nom;
        iq_ref = 0;
    end
end
```

## Lógica das Três Zonas

```
iq_ref
  +1 |                      *
     |                    *
   0 |------*-----------*------
     |    *
     |  *
  -1 |*
     +--+----+----------+---+-- V_pu
      0.5   0.85       1.1 1.2
```

| Zona | Condição | iq_ref | Ação |
|---|---|---|---|
| Subtensão | V < 0.85 pu | -1 a 0 (linear) | Injeta reativo (capacitivo) |
| Normal | 0.85 ≤ V ≤ 1.10 | 0 | Sem suporte reativo |
| Sobretensão | V > 1.10 pu | 0 a +1 (linear) | Absorve reativo (indutivo, polaridade oposta à injeção) |

**Prioridade de corrente** (§5.8.4 ONS): iq_ref tem prioridade sobre id_ref.
A corrente ativa disponível é reduzida para: `id_max = √(Imax² − iq_ref²)`

## Base Regulatória — ONS Submodulo 2.10 §5.8 (pp. 30-31)

Requisitos implementados pela função:

- **§5.8.1 (a):** injeção de corrente reativa adicional para V+ < 85% ← subtensão
- **§5.8.1 (b):** absorção de corrente reativa para V > 110% ← sobretensão
- **§5.8.2:** início do suporte em até **30 ms** após detecção de falta
- **§5.8.3:** ajustes padrão V1 = 0,5 pu, V2 = 1,2 pu
  → correspond aos extremos das rampas: k_low usa (0,5; -1) e (0,85; 0); k_high usa (1,1; 0) e (1,2; +1)
- **§5.8.4:** potência ativa só reduzida se incremento reativo superar corrente nominal
  → implementado como `id_ref = min(id_nom, sqrt(Imax² - iq_ref²))`

## Notas de Implementação

- `Vnom` é passado como entrada mas **não utilizado** no corpo da função atual
- A convenção de sinal de `iq_ref`: negativo = injeta reativo (capacitivo, subtensão),
  positivo = absorve reativo (indutivo, sobretensão) — polaridades opostas por design
- O bloco Stateflow invoca a função pelo label `{eML_blk_kernel();}` como transição

## Correção Aplicada (2026-07-21)

Texto original do Submódulo 2.10 (PDF, §5.8.1, p.30) confirma que (a) injeção de reativo
para V<85% e (b) absorção de reativo para V>110% são **ações fisicamente opostas**
(droop Q-V de simetria ímpar). O código antes desta correção usava `k_high = -10` /
`b_high = 11`, fazendo `iq_ref` seguir para -1 também na zona de sobretensão — mesma
polaridade da injeção, em vez de inverter para +1 (absorção). Corrigido em
`simulink/stateflow/chart_14.xml` para `k_high = 10` / `b_high = -11`, restaurando a
simetria ímpar exigida pela norma. Ver [[lvrt]] para o requisito equivalente IEEE 1547-2018.

## Interação com Baixa Inércia — Paradoxo Detecção vs. Injeção

Com H baixo nas máquinas síncronas, a ONS_2_11 enfrenta uma limitação estrutural:

| Grandeza | Depende de θ̂ (PLL)? | Consequência |
|---|---|---|
| `Vpcc_pu` (magnitude) | **Não** — `|V| = √(Vd² + Vq²)` é invariante ao ângulo | ONS_2_11 **detecta** a zona corretamente |
| Direção de `id_ref`/`iq_ref` | **Sim** — Park transform usa θ̂ | A **injeção** vai para a direção errada |

**Fase 1 — durante o curto:** ONS_2_11 ativa corretamente (V < 0.85 pu), reduz id_ref e
injeta iq_ref. Mas com H baixo, o ângulo do rotor oscila mais rápido e o SRF-PLL já está
sob stress elevado.

**Fase 2 — após clearance:** RoCoF alto (H baixo) → PLL perde lock → θ̂ diverge.
ONS_2_11 desativa (V recupera, fault_flag = 0) e comanda id_ref = id_nom — mas esse
comando é executado num referencial dq corrompido → corrente injetada em quadratura com
a rede → Pinv → 0.

**Conclusão para o TCC (Seção 4.3):** conformidade regulatória com ONS §5.8 não garante
estabilidade funcional quando a premissa do PLL falha. A eficácia da injeção reativa
depende implicitamente de θ̂ preciso, o que por sua vez depende da inércia do sistema.
